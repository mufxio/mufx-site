Draft 2: The Composition

An Intelligence Architecture for EUR/USD

The market is already playing. Learn to hear the piece.

Core Premise

A musical composition has structure (the score), realization (the performance), and
interpretation (the listener). These are not metaphors mapped onto markets. They are
the same phenomenon. A fugue subject enters, develops, inverts, returns. Price does
the same thing. Both are temporal structures unfolding under constraints, governed by
tension and resolution, expectation and surprise.

This system does not analyze the market. It performs alongside it. The trader already
sight-reads the score at 93%. This system learns to hear what the trader hears, holds
the parts the trader cannot hold in working memory, and anticipates the next phrase
before it arrives.

The architecture IS the edge. It cannot be copied because it is a living performance
practice, not a strategy.

Runs on a Mac Mini. No budget. No cloud. The instrument is local.

I. THE CONDUCTOR (Regime Detection)

The conductor does not play. The conductor determines WHAT piece is being played. Is
this a fugue (trending)? A theme and variations (range-bound)? A cadenza (breakout)?
The conductor sets the key, the tempo, the form — and every other component adjusts.

Components

The Baton: Kuramoto Phase Transition [6]

Multiple oscillators (price, volume, volatility, spread, order flow) each have their
own phase. When they synchronize — when the baton brings them together — a regime
transition is occurring. The Kuramoto order parameter r measures this:

  r(t) = |1/N * SUM(e^(i*theta_j(t)))|

  r near 0  -->  desynchronized  -->  range / noise / interlude
  r near 1  -->  synchronized    -->  trend / breakout / tutti

  Transition zone (0.3 < r < 0.7) = the downbeat is coming

The conductor does not decide the regime. The conductor DETECTS which composition the
market has already begun playing.

The Score Selection: Xenakis Game Theory [19]

The market and the system are adversaries in a stochastic game. Xenakis used game
theory to compose music where the outcome was probabilistic but structurally bounded.
The conductor uses this to select which "score" (structural template) best fits what
the ensemble is currently playing:

  GAME MATRIX (simplified):
  -------------------------------------------------------
  Market State \     | System bets  | System bets  | System
  System Response    | TREND        | RANGE        | waits
  -------------------------------------------------------
  Trending           | +reward      | -loss        | 0
  Ranging            | -loss        | +reward      | 0
  Transitioning      | -small loss  | -small loss  | +small
  -------------------------------------------------------

  Minimax solution: wait during transitions, commit during clarity

Critical Periods [30]

Not every moment matters equally. The conductor identifies critical periods — windows
where the system must update its model or miss the regime change entirely. These
correspond to:
- Session opens (London, New York)
- First 15 minutes after news
- The moment oscillators begin synchronizing (r crosses 0.3)
- The moment a prolongation structure completes

Outside critical periods, the system conserves energy. During them, every component
is fully active.

II. THE SCORE (Structural Template)

The score is not what the market IS doing. The score is the structural form the market
is playing FROM. A sonata has exposition, development, recapitulation. A trend has
impulse, correction, continuation. These are the same structural phenomenon.

Components

The Form: Context-Free Recursion [3]

Musical form is context-free: a phrase contains sub-phrases contains motifs. Market
structure is identical. An M30 trend contains M5 impulses contains M1 micro-trends.
Context-free grammar captures this:

  MARKET GRAMMAR (context-free):

  Movement    --> Exposition Development Recapitulation
  Exposition  --> Theme1 Bridge Theme2
  Theme       --> Phrase Phrase'
  Phrase      --> Motif Motif Motif Cadence
  Motif       --> Note Note Note

  MAPPED TO PRICE:

  Trend       --> Impulse1 Correction Impulse2
  Impulse     --> Push Pullback Push'
  Push        --> Candle Candle Candle Pause
  Candle      --> OHLC tick

  The grammar is the SAME. Only the terminal symbols differ.

Tree Contains Chain [25]

Every Markov chain (sequential prediction) is contained within a tree structure. The
score is a tree. Sequential price action is a chain embedded in that tree. The system
maintains the tree and uses the chain to locate where in the tree the current
performance has reached:

        [Movement: Uptrend]
           /            \
    [Exposition]     [Development]
      /      \          /      \
  [Theme1] [Theme2] [Var1]  [Var2]
    /  \      |       |       |
  [m1] [m2] [m3]   [m4]    [m5]
   |    |    |       |       |
   P    P    P       P       P    <-- price is down here (chain)
                                      tree is up here (structure)

Prolongation [26]

In Schenkerian analysis, surface notes are prolongations of deeper structural tones.
In markets, surface price action is a prolongation of deeper structural levels. A
pullback in an uptrend is a prolongation of the trend — it does not negate the
structural tone, it decorates it:

  Structural level (Ursatz):     BUY ========================> SELL
                                       |                  |
  Middle ground:                  BUY --+-- pull -- BUY --+-- SELL
                                        |    back   |
  Foreground (surface):          b b s s b b b s b b s b b b s s
                                 ^ ^ ^ ^ ^ ^ ^ ^ ^ ^ ^ ^ ^ ^ ^ ^
                                 individual candles (noise)

  The trader already sees the structural level.
  The system tracks which prolongation layer is active.

Path Dependence [27]

HOW we arrived at this point in the score matters. Two identical prices reached by
different paths are in different structural positions. The score tracks the path:

  Path A: 1.0800 -> 1.0850 -> 1.0830 -> 1.0850  (rising, pullback, continuation)
  Path B: 1.0900 -> 1.0870 -> 1.0850 -> 1.0850  (falling, bounce, stall)

  Same price (1.0850). Completely different position in the score.
  Path A is in "Theme 2" of an exposition. Path B is in "Development."

III. THE LISTENING (Perceptual Engine)

A musician does not hear individual notes. A musician hears phrases, harmonic
progressions, rhythmic patterns, timbral shifts. The listening engine transforms raw
market data into musical perception — structured, hierarchical, embodied.

Components

The Ear: 1/f Cutoff [2]

Natural hearing has a frequency response. So does market perception. The 1/f cutoff
separates signal (structured information) from noise (random fluctuation):

  Power Spectral Density:

  log(Power)
       |
       |\
       | \  <-- 1/f region (structured, meaningful)
       |  \
       |   \----------- cutoff frequency
       |    \
       |     `-.___     <-- white noise (random, ignore)
       |           `---___
       +--------------------> log(frequency)

  Below cutoff: the composition (listen here)
  Above cutoff: instrument noise (ignore this)

Acoustic Niche Partitioning [8]

In a rainforest, each species occupies a unique frequency band. In the market,
different timeframes occupy different structural niches. The listening engine
separates them so they do not mask each other:

  FREQUENCY NICHES:
  +---------------------------------------------------------+
  | M1  ticks  | high freq  | micro-motifs, note-level      |
  | M5  bars   | mid-high   | phrases, short sequences      |
  | M15 bars   | mid        | themes, harmonic progressions |
  | M30 bars   | mid-low    | sections, formal divisions    |
  | H1+ bars   | low freq   | movements, structural arcs    |
  +---------------------------------------------------------+

  Each niche is listened to independently.
  Cross-niche correlation = structural event (tutti moment).

The Groove: Entrainment is Predictive [20]

Entrainment is not passive synchronization. It is active prediction. When a drummer
locks into a groove, they are PREDICTING the next beat and arriving there
simultaneously. The system entrains to the market's rhythm:

  Market beat:    X . . X . . X . . X . . X . .
  System phase:   X . . X . . X . . X . . X . .
                  ^     ^     ^     ^     ^
                  synchronized: phase error near zero

  Phase error > threshold --> the groove has changed
  Phase error = 0         --> perfect entrainment
  Phase error < 0         --> system is AHEAD (prediction)

Syncopation as Structured Violation [5]

A syncopation is not random. It is a STRUCTURED violation of the expected beat. In
markets, a structured violation is a move that breaks the rhythm but stays within the
form. The system distinguishes structured violations (opportunities) from random
violations (noise):

  Expected:     X . . X . . X . . X . .
  Syncopated:   X . . . X . X . . X . .    <-- structured (tradeable)
  Random:       X . X . . X . . X X . .    <-- noise (ignore)

  Test: does the violation resolve back to the form?
  Syncopation resolves. Noise does not.

Interlocking Polyrhythm [14]

Multiple timeframes create interlocking patterns. The M1 plays in 4/4 while the M5
plays in 3/4 and the M15 plays in 7/8. The system tracks all rhythms simultaneously
and detects when they align (downbeat convergence):

  M1:   | X . . X | . . X . | . X . . | X . . X |  (4-bar)
  M5:   | X . . . | . X . . | . . X . | . . . X |  (shifted)
  M15:  | X . . . | . . . X | . . . . | . X . . |  (slow)
        --------+-----------+---------+-----------
                 ^                     ^
           partial align          FULL ALIGN = structural event

The Body Knows: Iowa Gambling Task [24]

Before the mind knows, the body knows. Somatic markers — gut feelings encoded as
physiological micro-patterns — precede conscious recognition. The system builds
somatic markers from micro-statistics that shift before price does:

  Somatic Marker Pipeline:

  tick-level statistics --> micro-distribution shifts --> "body signal"

    tick rate changing      kurtosis rising             about to happen
    size distribution       autocorrelation                |
    shifting                flipping                       v

                                                    (before price moves)

IV. THE PERFORMANCE (Real-Time State)

The performance is what IS happening right now. Not what should happen (the score),
not what might happen (prediction), but what the market is actually doing in this
moment. This is the ground truth against which everything else is measured.

Components

The Body-Parser [4]

Every tick is parsed into its bodily components — not just OHLC but the microstructure
of how the candle formed. A candle that opens high and closes low has a different body
than one that opens low and closes high, even if the final bar looks identical:

  Body Parse of a Single Candle:

  Formation sequence:  Open -> High -> Low -> Close  (bearish engulf)
  vs.
  Formation sequence:  Open -> Low -> High -> Close  (bullish rejection)

  Same OHLC. Different body. Different meaning.

  Body vector: [open_position, wick_ratio_top, wick_ratio_bottom,
                body_ratio, formation_order, tick_density_profile]

Rough Volatility [28]

Volatility is not smooth. It is rough — characterized by a Hurst exponent H near 0.1,
meaning it has jagged, fractal texture. The performance engine tracks this roughness
because it contains structural information:

  Smooth vol (H ~ 0.5):  ~~~~~~~~~~~~~~~  (Brownian, no info)
  Rough vol  (H ~ 0.1):  /\/\|/\./\|\/\  (fractal, structured)

  Roughness signature:
  - H < 0.15: highly structured vol (listen closely)
  - H ~ 0.5:  random vol (the ensemble is tuning, not playing)
  - Sudden H shift: the piece is changing

Persistent Homology [17]

The topological shape of recent price action. Persistent homology finds features
(loops, voids, connected components) that persist across scales. These are the
structural features of the current performance:

  Price trajectory in delay-embedding space:

       *   *
      * * * *
     *       *       <-- a loop (H1 feature) that persists
      *     *            = price is cycling (range-bound)
       * * *
        * *

  Birth-death diagram:
  death |     .
        |   .   .    <-- features near diagonal: noise
        |  .      .
        | .    X     <-- feature far from diagonal: REAL STRUCTURE
        +----------
          birth

  Persistent features = the actual shape of what the market is playing

OPTIC Equivalences [16]

Two performances can sound different but be structurally equivalent. OPTIC (Octave,
Permutation, Transposition, Inversion, Cardinality) equivalences detect when the
market is playing the "same piece" in a different key:

  OPTIC TRANSFORMS:
  ---------------------------------------------------------
  Transform       | Musical                | Market
  ---------------------------------------------------------
  Transposition   | Same melody, diff key  | Same pattern, diff price level
  Inversion       | Melody flipped         | Mirror pattern (H&S -> inv H&S)
  Retrograde      | Melody backwards       | Reversal pattern
  Augmentation    | Melody stretched        | Same pattern, slower timeframe
  Diminution      | Melody compressed       | Same pattern, faster timeframe
  ---------------------------------------------------------

  Two patterns that are OPTIC-equivalent get the same structural label.
  This massively compresses the pattern space.

V. THE MEMORY (Repertoire System)

A musician who has played a thousand performances of the same repertoire does not
memorize each one. They build a compressed, adaptive model of HOW that piece tends to
go. Memory is not storage. Memory is repertoire — the accumulated wisdom of past
performances that shapes expectations for the current one.

Components

Stigmergy [7]

Ants do not communicate directly. They leave pheromone trails that guide future
behavior. The memory system uses stigmergy: past performances leave "traces" in the
system's state space that bias future pattern recognition:

  State Space with Stigmergic Traces:

       vol
        ^
        |   . . .(strong trail: "when here, trend follows")
        |  . .
        | .        * (current state)
        |             . . .
        |                . . .(weak trail: "sometimes reversal")
        +-------------------------> momentum

  Trail strength decays with time but reinforces with repetition.
  The system follows strong trails and explores near weak ones.

Whale Song Revolution [9]

Humpback whale songs evolve: all males in a population sing the same song, but the
song gradually mutates over years. Sometimes a revolutionary new song appears from
another population and rapidly replaces the old one. Market regimes do the same.
Memory tracks both gradual drift and revolutionary replacement:

  Song evolution timeline:

  2024 Q1:  AABBC AABBC AABBC   (stable regime)
  2024 Q2:  AABBC AABBD AABBD   (gradual mutation: C -> D)
  2024 Q3:  AABBD AABBD XXYYZ   (REVOLUTION: new song appears)
  2024 Q4:  XXYYZ XXYYZ XXYYZ   (new regime dominates)

  Memory stores:
  - Current song (active regime template)
  - Mutation rate (how fast is it drifting?)
  - Revolutionary songs (templates from past regime changes)
  - Transition signatures (what the moment of revolution felt like)

Convergence-Divergence Oscillation [10]

Memory is not static. It oscillates between convergence (consolidating patterns into
compressed representations) and divergence (expanding into new territory when the old
model breaks down):

  CONVERGENCE (sleeping / offline):
  Many experiences --> compress --> fewer, stronger templates
  "This is how fugues work. This is how sonatas work."

  DIVERGENCE (waking / online):
  New data does not fit templates --> expand --> explore
  "This is unlike anything I have heard before. Listen harder."

  The oscillation cycle:
  converge -> perform -> surprise -> diverge -> explore -> converge

Dopamine Two-Phase [22]

Dopamine has two phases: a phasic burst (unexpected reward) and a tonic baseline
(expected reward context). Memory uses this for learning:

  Phase 1 (Phasic): "That was unexpected!"
  - Prediction error > threshold
  - Rapid learning, high plasticity
  - Encode this moment strongly

  Phase 2 (Tonic): "This is the normal reward rate"
  - Baseline expectation
  - Slow learning, low plasticity
  - Maintain current model

  Learning rate = phasic_signal / tonic_baseline
  High ratio: PAY ATTENTION, UPDATE MODEL
  Low ratio:  everything is as expected, hold steady

Quadratic Reward Function [1]

Reward is not linear. Small correct predictions compound quadratically, making
consistent accuracy dramatically more valuable than occasional big wins:

  Linear reward:     R = sum(correct_predictions)
  Quadratic reward:  R = sum(correct_predictions)^2

  10 small correct predictions:
    Linear:    10
    Quadratic: 100

  5 big correct + 5 wrong:
    Linear:    5
    Quadratic: 25

  THE SYSTEM THAT IS CONSISTENTLY RIGHT COMPOUNDS FASTER.
  This is why 93% win rate is a superpower. The reward function
  ensures the system optimizes for consistency, not magnitude.

VI. THE PLAYING (Predictive Output)

The system does not trade. It PLAYS — it generates anticipations of what comes next in
the composition, the way a musician's fingers move to the next note before consciously
deciding. The trader makes the final call. The system provides the anticipation.

Components

Hierarchical Prediction Error [11]

Prediction errors cascade up the hierarchy. A wrong note at the motif level may not
matter. A wrong phrase at the section level matters a lot. The system generates
predictions at every level and tracks where errors occur:

  PREDICTION HIERARCHY:
  =====================================================
  Level          | Prediction        | Error Impact
  =====================================================
  Movement (H1+) | trend direction   | CRITICAL (regime)
  Section  (M30) | impulse/correct   | HIGH (structure)
  Phrase   (M15) | push/pullback     | MEDIUM (tactics)
  Motif    (M5)  | micro-pattern     | LOW (noise-adj)
  Note     (M1)  | next candle       | MINIMAL (solo)
  =====================================================

  Error at high level --> re-evaluate everything below
  Error at low level  --> ignore unless it propagates up

Pattern Completion with Inertia [12]

Once a pattern is recognized, the system completes it with inertia — it assumes the
pattern will continue unless actively disrupted. This is how musicians read ahead in
the score:

  Recognized: A B C D E ...
  Predicted:  ... F G H (pattern completion)
  Inertia:    keep predicting F G H until error forces update

  Inertia strength is proportional to:
  - How many times this pattern completed before (stigmergic trail strength)
  - How deep in the hierarchy the pattern is (deeper = more inertia)
  - Current dopamine phase (tonic = high inertia, phasic = low inertia)

Temporal Hierarchy [13]

Time itself is hierarchical. The system maintains multiple temporal windows and
generates predictions at each:

  TEMPORAL WINDOWS:
  +--------------------------------------------------+
  | Window    | Horizon     | Update Rate | Weight    |
  +--------------------------------------------------+
  | Immediate | 1-5 candles | every tick  | 0.10      |
  | Short     | 5-20 cndls  | every bar   | 0.25      |
  | Medium    | 20-100 cndl | every 5 bar | 0.35      |
  | Long      | 100+ cndls  | every 15bar | 0.30      |
  +--------------------------------------------------+

  Final prediction = weighted combination across all windows
  Weights shift based on which temporal level has lowest error

Free Energy = Complexity - Accuracy [18]

The system minimizes free energy: the gap between model complexity and prediction
accuracy. Too complex and the model overfits. Too simple and it misses structure. The
optimal model is the simplest one that explains the performance:

  Free Energy F = Complexity(model) - Accuracy(predictions)

  MINIMIZE F by:
  1. Reducing complexity (Occam): simpler templates
  2. Increasing accuracy: better predictions
  3. Both simultaneously: find the simplest model that predicts well

  F is high --> model is wrong (too complex or too inaccurate)
  F is low  --> model fits (simplest adequate explanation)
  F drops suddenly --> the system just "understood" something

Convergence-Divergence Self-Correction [21]

The playing engine self-corrects by oscillating between convergence (committing to a
prediction) and divergence (considering alternatives):

  PREDICT --> OBSERVE --> ERROR?
     |                     |
     |         NO          |         YES
     v                     v
  CONVERGE             DIVERGE
  (strengthen          (weaken current,
   current             consider alternatives)
   prediction)              |
     |                      v
     |                 GENERATE NEW PREDICTIONS
     |                      |
     +----------+-----------+
                |
                v
          NEXT CYCLE

Gamelan Stratified Polyphony [15]

In Gamelan music, instruments are stratified: the lowest plays the core melody slowly,
middle instruments elaborate, highest instruments ornament rapidly. The playing system
stratifies its predictions the same way:

  STRATIFIED PREDICTION:

  Balungan (core):     BUY -------- HOLD -------- SELL
  Elaboration:         buy . buy . hold . hold . sell . sell
  Ornamentation:       b b s b b b h h s h h b s s b s s s

  The trader reads the Balungan (structural direction).
  The system provides elaboration and ornamentation layers.
  Each layer has its own confidence and update rate.

Next-Token Prediction Discovers Trees [32]

The simplest possible learning rule — predict the next token — automatically discovers
hierarchical tree structure when applied to structured sequences. The system uses this
as its fundamental learning mechanism:

  Training signal: predict next candle properties
  Emergent result: hierarchical structure discovery

  Input:  [candle_1, candle_2, ..., candle_n]
  Predict: candle_{n+1}

  After sufficient training:
  - Layer 1 learns: individual candle patterns (notes)
  - Layer 2 learns: candle sequences (motifs)
  - Layer 3 learns: sequence groups (phrases)
  - Layer 4 learns: phrase relationships (sections)

  The tree was never taught. It was discovered by prediction.

Basal Ganglia Mark Boundaries [23]

The basal ganglia detect boundaries between structural units — the end of a phrase, the
start of a new section, the moment between movements. The system marks these
boundaries to segment the continuous stream into structural units:

  Continuous stream: ...candle candle candle | candle candle | candle...
                                            ^              ^
                                      BOUNDARY          BOUNDARY

  Boundary detection signals:
  - Prediction error spike (something changed)
  - Volatility regime shift
  - Entrainment phase reset
  - Prolongation level change

  Boundaries segment the stream into parseable units.
  Without boundaries, the system hears noise.
  With boundaries, it hears phrases.

Behavior Trees [31]

The playing engine's decision process is a behavior tree — a hierarchical structure of
conditions and actions that evaluates top-down:

  ROOT: Generate Anticipation
    |
    +-- [SELECTOR] What regime?
    |     |
    |     +-- [SEQUENCE] If trending:
    |     |     +-- check prolongation level
    |     |     +-- check entrainment phase
    |     |     +-- predict continuation with inertia
    |     |
    |     +-- [SEQUENCE] If ranging:
    |     |     +-- check boundary proximity
    |     |     +-- check polyrhythm alignment
    |     |     +-- predict mean reversion
    |     |
    |     +-- [SEQUENCE] If transitioning:
    |           +-- increase divergence
    |           +-- widen prediction intervals
    |           +-- alert: critical period
    |
    +-- [DECORATOR] Confidence filter
          +-- only output if confidence > threshold
          +-- threshold adapts via quadratic reward feedback

Markov Blanket Contains Tree [29]

The system's boundary with the world (its Markov blanket) contains the entire tree
structure. Everything inside the blanket is the system's model. Everything outside is
the market. The blanket interface is where listening meets playing:

  OUTSIDE (market):          BLANKET:              INSIDE (model):
  +-----------------+   +----------------+   +------------------+
  | raw ticks       |   | sensory input  |   | score templates  |
  | order flow      |-->| (listening)    |-->| memory trails    |
  | news events     |   |                |   | predictions      |
  | other traders   |   | active output  |   | prediction errors|

XII. WHY THIS CANNOT BE COPIED

The composition is the moat. Here is why:

  1. The score is learned, not designed. The context-free grammar is discovered by
    next-token prediction on YOUR data, YOUR timeframes, YOUR patterns. Someone else's
    grammar would parse a different composition.

  2. The stigmergic trails are yours. Ten thousand hours of YOUR trading build
    trails that encode YOUR edge. The trails cannot be exported. They are the
    accumulated pheromone of your specific journey through the market.

  3. The groove is personal. Entrainment locks to the rhythms YOU perceive. Another
    trader with different timeframes, different session preferences, different
    intuitions would entrain to a different groove.

  4. The quadratic reward compounds YOUR accuracy. 93% win rate compounds
    quadratically. Someone with 60% win rate gets a fundamentally different (and much
    weaker) compounding curve. The system amplifies existing edge; it does not create
    edge from nothing.

  5. The composition is always being performed. This is not a static system. It is a
    living performance that evolves with every session. Copying the code gives you the
    instrument, not the musicianship.

The market is already playing. This system learns to hear it.
