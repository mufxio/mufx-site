Verification

Publication Integrity

Every muFX article carries a SHA-256 hash of its content, generated at publication and committed to a public Git repository. This creates an immutable, timestamped record proving exactly what was published and when.

What the hash proves

The hash in each article header is a cryptographic fingerprint of the article's text content. If a single character changes after publication, the hash no longer matches. The Git commit timestamp is the proof of when the content existed in that exact form.

This means no retroactive edits, no quiet corrections, no claiming calls that weren't made. The record is public and independently verifiable.

How to verify

01
Find the hash

Click the hash code in any article header. It looks like: a1b2c3d4e5f6

02
Check the commit

Visit the public repository and find the commit that published the article. The commit date is the publication timestamp.

03
Reproduce the hash

Clone the repo, extract the article text from the HTML, and run echo -n "$TEXT" | shasum -a 256. If the output matches the hash in the article, the content is unchanged from publication.

Why this matters

FX research has a credibility problem. Analysts revise calls after the fact, delete bad predictions, and selectively highlight wins. muFX publishes with cryptographic proof specifically so this isn't possible. Every call is on the record, permanently.
