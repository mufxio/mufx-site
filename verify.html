Verification

Publication Integrity

Every muFX article carries a SHA-256 hash of its content, generated at publication and committed to a public Git repository. This creates an immutable, timestamped record proving exactly what was published and when.

How it works

01
Content is hashed at publication

When an article is published, the text content inside the <article> tag is extracted, HTML tags are stripped, whitespace is normalized, and a SHA-256 hash is computed. This hash is embedded in the page and committed to a public Git repository.

02
The hash is a fingerprint

SHA-256 produces a unique 64-character string for any given input. If a single character of the article changes after publication, the hash will no longer match. No retroactive edits, no quiet corrections, no claiming calls that weren't made.

03
Anyone can verify

The tool above fetches the article, extracts the text, computes SHA-256 in your browser using the Web Crypto API, and compares it to the published hash. Everything runs client-side — nothing is sent to a server.

Why this matters

FX research has a credibility problem. Analysts revise calls after the fact, delete bad predictions, and selectively highlight wins. muFX publishes with cryptographic proof specifically so this isn't possible. Every call is on the record, permanently.
