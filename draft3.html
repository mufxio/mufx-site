Draft 3: The Growing Tree

The Architecture That Grows Itself

This is the most literal of the three drafts. The system IS a tree -- not a metaphor, not an analogy, a literal tree data structure that grows as the system processes EUR/USD market data. It starts empty. It encounters price action. It grows. What it becomes after 20 years of growth reveals what the euro IS -- the same way a tree's shape reveals the wind patterns, soil composition, and light conditions of the place where it grew.

Your 93% win rate by sight-reading M1-M30 charts is the proof that the euro HAS a nature. This tree will grow into the shape of that nature. Your edge is not replaced -- it is the sunlight that tells the tree which direction to grow.

I. THE SEED: What a Node Contains

Every node in the tree is a state snapshot -- a compressed representation of "what the market looks like right now." The tree is a collection of these snapshots connected by parent-child edges that encode "what happened next."

Node Data Structure

Node {
    id:             u64                  // unique, monotonic
    depth:          u16                  // distance from root

    // --- STATE (what the market looks like here) ---
    state:          StateVector {
        price_ratios:   [f32; 8]         // price relative to N lookbacks
        volatility:     f32              // realized vol (roughness-aware)
        spread_regime:  u8               // 0-3: tight/normal/wide/blown
        momentum:       [f32; 4]         // slope at 4 timescales
        rhythm:         [f32; 6]         // autocorrelation at lag 1-6
        body_signal:    f32              // aggregate somatic marker
        phase:          f32              // 0..2pi Kuramoto phase angle
        blanket:        [f32; 4]         // Markov blanket boundary vars
    }

    // --- STATISTICS (what has happened from here) ---
    outcomes:       OutcomeStats {
        count:          u32              // times we have been here
        mean_return:    f32              // average next-N return
        variance:       f32              // variance of returns
        win_rate:       f32              // fraction positive
        max_drawdown:   f32              // worst path from here
        reward_surface: [f32; 16]        // quadratic reward at 16 horizons
    }

    // --- GROWTH METADATA ---
    created_at:     u64                  // first observation timestamp
    last_visited:   u64                  // most recent observation
    visit_count:    u32                  // total traversals
    committed:      bool                 // past critical period? frozen?
    ring:           u16                  // which growth year
    branch_reason:  Option<String>       // what dimension caused the split

    // --- CONNECTIONS ---
    parent:         Option<NodeId>
    children:       Vec<NodeId>
    leaf_preds:     Vec<Prediction>      // active predictions (leaves only)
}

What This Means Concretely

The state vector is what the tree uses to determine "where am I?" when new data arrives. The outcomes accumulate what happened every time the system was at this node. The committed flag is the critical period -- once a node has enough evidence, its state boundaries freeze and it becomes structural (trunk or major branch). Uncommitted nodes are still growing, still negotiable.

II. THE ROOT: Day Zero

The tree starts as a single node. It knows nothing. Every observation maps to the same place.

         [ROOT]
     state: null
     outcomes: empty
     committed: false
     ring: 0
     depth: 0

The first price tick arrives. The root records it. The second tick arrives. The root records the return. After 1000 ticks, the root has outcome statistics but no branches. It can make one prediction: the unconditional distribution of EUR/USD returns.

This IS the seed. It is alive but undifferentiated.

Mechanic 30 (Critical periods): The root itself will commit early -- its unconditional statistics stabilize within weeks. Once committed, the root's state boundaries are permanent. It becomes the trunk's base. This is the first critical period closure.

III. THE FIRST BRANCH: Disagreement Creates Structure

After accumulating enough observations, the root's children disagree. The branching condition is precise:

Branching Condition

BRANCH when:
    node.visit_count > MIN_VISITS (e.g., 500)
    AND
    nearest_neighbor_disagreement(node) > THRESHOLD

where nearest_neighbor_disagreement =
    max over dimension d of:
        | mean_return(observations where state[d] > median)
          - mean_return(observations where state[d] < median) |
        / pooled_stddev

In plain language: look at all the times the system was at this node. Split them along each dimension of the state vector. Find the dimension where the two halves disagree most about what happens next. If the disagreement is large enough, that dimension becomes a branch point.

Mechanic 25 (Tree contains chain): Every path from root to leaf is a Markov chain. The tree structure means multiple chains share prefixes. A chain is just a tree with branching factor 1. The tree contains all chains as degenerate paths.

The First Split in Practice

After 2 weeks of M1 data (~14,400 observations):

                    [ROOT]
                 committed: true
                 ring: 0
              /              \
    [LOW VOL]                  [HIGH VOL]
    vol < 0.04                 vol >= 0.04
    committed: false           committed: false
    ring: 0                    ring: 0
    mean_ret: +0.1bp           mean_ret: -0.3bp
    win_rate: 0.54             win_rate: 0.47

The tree has discovered its first structural fact about the euro: low volatility and high volatility states have different return profiles. This is trivial. Every quant knows this. But the tree discovered it from nothing, the way a seedling discovers that light comes from above.

Mechanic 11 (Hierarchical prediction error): The root predicted one distribution. Now the children predict two different distributions. The prediction error at the root -- the gap between its prediction and reality -- is what drove the split. Each child now has lower prediction error than the parent did. The hierarchy is: root predicts coarsely, children predict more precisely.

IV. GROWTH MECHANICS: How the Tree Develops

Year 1: Root System and Trunk

MONTH 1-3: Rapid shallow branching

                           [ROOT]
                        committed: true
                     /         |         \
              [LOW VOL]    [MED VOL]    [HIGH VOL]
              committed     growing      committed
             /    \           |          /    \
         [trend+] [trend-]  [flat]   [spike] [grind]
         growing   growing  growing  growing  growing

    ~20-50 nodes
    Depth: 3-4
    Most branches uncommitted
    Ring 0

Mechanic 13 (Temporal hierarchy): The tree naturally develops temporal layers. The root splits on slow-moving features (volatility regime). The next level splits on medium-speed features (trend direction). Deeper levels split on fast features (tick-level rhythm). This is not designed -- it emerges because slow features explain the most variance at the top and fast features only matter once you have conditioned on the slow context.

Mechanic 6 (Kuramoto phase transition): Multiple state dimensions oscillate independently at first. When enough data accumulates, their phases lock -- the tree discovers that certain combinations of features always appear together. This phase-locking is what commits a node. The oscillators have synchronized; the state boundary is stable.

Year 1-2: Branch Differentiation

MONTH 6-12: Structural branches commit, surface stays fluid

                              [ROOT]
                           committed, ring 0
                    /           |            \
            [LOW VOL]       [MED VOL]      [HIGH VOL]
            committed       committed       committed
            ring 0          ring 0          ring 0
           /    \              |            /    \
      [up]      [dn]       [range]    [spike]  [grind]
      comm.     comm.       comm.      comm.    comm.
      ring 1    ring 1      ring 1     ring 1   ring 1
     / | \      / \         / | \       |       / \
   (leaves)  (leaves)   (leaves)    (leaves) (leaves)
   growing   growing    growing     growing  growing
   ring 1    ring 1     ring 1      ring 1   ring 1

    ~200-500 nodes
    Depth: 5-8
    Trunk and major branches committed
    Ring 1 visible in committed nodes

Mechanic 2 (1/f cutoff): The tree's branching frequency follows a 1/f distribution. Major branches (regime changes) are rare and deep. Fine branches (micro-pattern variations) are frequent and shallow. The cutoff determines the minimum timescale the tree bothers to model -- below this, it is noise.

Mechanic 26 (Prolongation): Some states are "prolongations" of their parents -- structurally subordinate, elaborations rather than departures. The tree encodes this: a prolongation node has nearly identical outcome statistics to its parent. It exists only to refine timing, not to change the prediction. These nodes are leaves that elaborate but do not contradict.

Year 2-5: Canopy Formation

                              [ROOT]
                           ring 0, committed
                    /           |            \
            [LOW VOL]       [MED VOL]      [HIGH VOL]
            ring 0-1        ring 0-1        ring 0-1
            committed       committed       committed
           /    \              |            /    \
      [trend+] [trend-]   [range]     [spike]  [grind]
      committed committed  committed   committed committed
     /  |  \    /  |  \    / | \       /  \     / | \
    ... ... .. .. .. ..  .. .. ..    ..   ..  .. .. ..

    ~2000-10000 nodes
    Depth: 8-15
    Clear trunk, branches, sub-branches
    Multiple rings visible
    CANOPY SHAPE EMERGES:

    Some branches are DENSE (many children, many leaves)
      = regimes the market spends a lot of time in
    Some branches are SPARSE (few children, tall)
      = rare but distinctive regimes
    Some branches are DEAD (committed, no recent visits)
      = regimes the market used to visit but no longer does

Mechanic 9 (Whale song revolution): The canopy shape changes over years. Early growth patterns get replaced. What was a major branch in year 1 might be a dead branch in year 5. The market's "song" changed -- its statistical signature shifted. The tree records this: dead branches remain (rings preserve history) but new growth happens elsewhere.

Mechanic 18 (Persistent homology): The tree's topological features (loops, voids, connected components at different threshold levels) persist across scales. A pattern that appears in the M1 canopy also appears in the M15 canopy. Persistent homology identifies which structural features are "real" (persist across scales) vs "noise" (appear at one scale only). The tree prunes noise-scale branches more aggressively.

V. THE MECHANICS: Where Each One Lives in the Tree

A. Reward and Learning (The Sap)

Mechanic 1 (Quadratic reward function):
The reward at each leaf is not linear in prediction accuracy. It is quadratic:

reward(node) = -alpha * (predicted_return - actual_return)^2
               + beta * confidence_when_correct^2
               - gamma * confidence_when_wrong^2

This means: being confidently wrong is MUCH worse than being unconfidently wrong. Being confidently right is much better than being unconfidently right. The tree learns to be confident only where it has evidence.

This lives in node.outcomes.reward_surface -- a 16-element array storing the quadratic reward at 16 different prediction horizons.

Mechanic 22 (Dopamine two-phase):
When a prediction resolves:
- Phase 1 (immediate, ~1 tick): prediction error signal. Positive if better than expected, negative if worse. This updates the parent node's branching thresholds.
- Phase 2 (delayed, ~10-50 ticks): did the CONTEXT of the prediction also improve? Did the tree's position in the state space become more precise? This updates the grandparent and higher nodes.

Event: leaf predicts +2 pips, actual = +5 pips

Phase 1: prediction_error = +3 pips (better than expected)
    -> parent node lowers branching threshold
    -> "this region is worth exploring more finely"

Phase 2 (50 ticks later): did being at this branch help?
    -> compare reward to what the grandparent would have predicted
    -> if yes: reinforce the branch dimension
    -> if no: this branch may be over-fitting

Mechanic 17 (Free energy = complexity - accuracy):
The tree's free energy is literally:

F(tree) = complexity(tree) - accuracy(tree)

complexity = total_nodes * log(total_nodes) + sum(node.depth * node.visit_count)
accuracy   = -sum over leaves of (visit_count * mean_squared_error)

Growth is justified only when adding a node reduces free energy. A new branch must improve accuracy more than it increases complexity. This is the tree's immune system against over-fitting.

B. Rhythm and Timing (The Rings)

Mechanic 5 (Syncopation as structured violation):
The tree expects certain rhythms in the data -- periodic patterns at session opens, fixes, option expiries. Syncopation occurs when the actual pattern violates the expected rhythm in a STRUCTURED way (not random). The tree detects this:

If:  expected_periodicity(node) is strong
And: actual_sequence violates it
And: the violation is itself periodic (meta-rhythm)
Then: create a new branch dimension: "syncopated"

This captures phenomena like: "the London fix usually pushes price one way, but on month-end it pushes the other way." The syncopation branch distinguishes month-end from normal days.

Mechanic 14 (Interlocking polyrhythm):
Multiple timescales produce predictions simultaneously. The M1 tree, M5 tree, and M15 tree each produce a prediction. These interlock:

M1  prediction:  +-+-+-+-+-+-+-+-    (fast oscillation)
M5  prediction:  +---+---+---+---    (medium cycle)
M15 prediction:  +-------+-------    (slow trend)

Interlocked:     When all three align = HIGH CONFIDENCE
                 When they disagree  = BRANCH POINT (ambiguity)

The tree does not run at one timescale. It is a FOREST of trees at multiple scales, and their polyrhythmic alignment IS the confidence signal.

Mechanic 15 (Gamelan stratified polyphony):
Like a gamelan ensemble, each layer of the tree plays a different "part":

Layer 0 (Root):     Fundamental tone -- regime identity
                    Changes slowest, anchors everything

Layer 1 (Trunk):    Elaboration -- trend within regime
                    Changes at medium speed

Layer 2 (Branch):   Ornamentation -- micro-pattern within trend
                    Changes fastest, most detailed

Prediction = all layers sounding simultaneously
           = the "chord" of current market state

Mechanic 23 (Basal ganglia mark boundaries):
Regime transitions -- moments when the tree's current leaf becomes inappropriate and the system must re-traverse from a higher node -- are boundary events. The system detects boundaries:

Boundary detected when:
    current_leaf.prediction_error > 3 * current_leaf.historical_stddev
    for N consecutive observations

Action:
    Ascend tree to nearest uncommitted ancestor
    Re-descend using current state
    Mark the transition as a boundary event
    Increment boundary_count on both the old and new branches

This is how the tree learns regime transitions: not by predicting them, but by detecting when its current position becomes wrong and recording where it re-enters.

C. Information Flow (The Roots)

Mechanic 7 (Stigmergy):
Nodes communicate indirectly through the data, not through direct message-passing. When a leaf node updates its outcome statistics, it does not notify other nodes. Instead, the next traversal of the tree will encounter the updated statistics and behave differently. The tree's "intelligence" is stigmergic -- it emerges from many simple local updates, not from a central controller.

Stigmergic trace:
    Leaf L updates: mean_return = -2bp (was +1bp)
    Nothing else happens immediately

    Next traversal reaches L's parent:
        Parent re-evaluates branching: L's sibling now looks better
        Traffic shifts away from L toward sibling
        L's visit_count decreases
        Eventually L may be pruned

    The "signal" traveled up the tree without any explicit propagation.

Mechanic 8 (Acoustic niche partitioning):
Branches at the same depth must partition the state space without overlap. Like species in a rainforest dividing the acoustic spectrum, sibling nodes divide the feature space so each occupies a unique niche:

Parent: [MED VOL]
    Child A: momentum > 0, rhythm[1] > 0.3    (trending, autocorrelated)
    Child B: momentum > 0, rhythm[1] < 0.3    (trending, mean-reverting)
    Child C: momentum < 0, rhythm[1] > 0.3    (declining, autocorrelated)
    Child D: momentum < 0, rhythm[1] < 0.3    (declining, mean-reverting)

No two children respond to the same input.
Every input maps to exactly one child.
The niches are non-overlapping and exhaustive.

Mechanic 29 (Markov blanket contains tree):
Each node has a Markov blanket -- the set of variables that, if known, render the node conditionally independent of everything outside. The blanket is stored in state.blanket:

node.state.blanket = [parent_state, children_states, direct_inputs]

If you know the blanket, you know everything the node needs to know.
The rest of the universe is irrelevant.

The TREE ITSELF is contained within a Markov blanket:
    External: raw price feed, news, order flow
    Blanket:  the state variables the tree tracks
    Internal: node statistics, tree structure

The blanket determines what the tree CAN learn.
Expanding the blanket (adding new state dimensions) = growing new roots.

D. Prediction and Navigation (The Canopy)

Mechanic 12 (Pattern completion with inertia):
When the current state partially matches a node, the tree completes the pattern using the node's full state vector. But with inertia -- the completion is biased toward the previous state:

current_partial_state = [vol=0.03, momentum=?, rhythm=[0.4, ?, ?, ?, ?, ?]]

Nearest node has:       [vol=0.03, momentum=+0.2, rhythm=[0.4, 0.3, 0.1, 0.05, 0.02, 0.01]]

Completed state:        [vol=0.03, momentum=+0.15, rhythm=[0.4, 0.28, ...]]
                                          ^--- blended with previous momentum (inertia)

Inertia prevents the tree from jumping between distant branches on noisy data. It takes sustained evidence to change the tree's mind about where it is.

Mechanic 20 (Entrainment is predictive):
The tree does not just react to data. When it has high confidence in its current position, it PREDICTS the next several ticks and pre-traverses the tree. This is entrainment -- the system locks onto the market's rhythm and runs slightly ahead:

Current position: [LOW VOL -> trend+ -> momentum_rising]
Prediction: next 10 ticks will continue rising
Pre-traverse: identify the leaf node 10 ticks ahead

If the market follows the predicted path:
    the system was already there, zero latency

If the market deviates:
    boundary detection fires (mechanic 23)
    re-traverse from higher node
    entrainment breaks, system re-locks

Mechanic 32 (Next-token prediction discovers trees):
The tree IS what you get when you do next-token prediction honestly. Each node predicts the next state. Where the prediction is ambiguous, the node splits. The tree structure is the natural output of recursive next-state prediction with branching on disagreement.

next_token_prediction(state) {
    node = traverse_tree(root, state)
    if node.is_leaf:
        return node.outcomes.mean_return, node.outcomes.variance
    else:
        // should not happen -- traverse always reaches a leaf
        // unless the tree is still growing at this branch
        return weighted_average(child.predict() for child in node.children)
}

This is the deepest mechanic: the tree is not an arbitrary choice. It is the NECESSARY structure that emerges from predicting the next state, noticing when predictions fail, and splitting on the failure mode.

E. Adaptation and Memory (The Bark)

Mechanic 3 (Context-free recursion):
The tree's growth algorithm is context-free: the same branching rule applies at every depth. A node at depth 2 branches by the same mechanism as a node at depth 12. The recursion is:

grow(node):
    if should_branch(node):
        dim = find_best_split_dimension(node)
        left, right = split(node, dim)
        grow(left)
        grow(right)
    if should_commit(node):
        node.committed = true
    if should_prune(node):
        remove(node)

This context-free recursion means the tree is self-similar. Zoom into any branch and it looks like a smaller version of the whole tree. Fractal structure from a trivial algorithm.

Mechanic 10 (Convergence-divergence oscillation):
The tree oscillates between two modes:

CONVERGENCE phase:
    Many branches collapse to similar predictions
    The tree "agrees with itself"
    Predictions are confident
    Growth slows

    Triggered when: inter-branch variance < threshold

DIVERGENCE phase:
    Branches disagree wildly
    The tree is "confused"
    Predictions are uncertain
    Growth accelerates (many new branches)

    Triggered when: inter-branch variance > threshold

The oscillation IS the learning rhythm.
Converge -> exploit knowledge -> diverge -> explore new structure -> converge

Mechanic 21 (Convergence-divergence self-correction):
When the tree has converged too far (over-confident) and then gets surprised:

State: tree is converged, confident, predicting +1bp
Event: actual return = -15bp (massive miss)

Self-correction sequence:
    1. Boundary detection fires at multiple levels
    2. Multiple committed nodes get un-committed (unprecedented)
    3. The trunk itself may sprout new branches
    4. This is a "growth spurt" -- structural re-organization

This is rare (maybe 2-3 times per year).
It corresponds to genuine regime changes.
The tree's rings record these events as unusually dense growth bands.

Mechanic 27 (Path dependence):
The tree's current structure depends on the ORDER in which data arrived. A tree grown on 2020 data then 2021 data is structurally different from one grown on 2021 then 2020. This is a feature, not a bug. The tree's path dependence means:

Tree grown 2015-2025:
    Deep branches for: ECB QE, negative rates, COVID vol
    These are committed, structural
    Recent branches: post-COVID normalization

Tree grown 2020-2025:
    No memory of pre-COVID regimes
    Different trunk structure entirely

The longer-lived tree is MORE VALUABLE because it has seen more.
Its path dependence IS its knowledge.
Its rings ARE its memory.

F. Structure and Topology (The Heartwood)

Mechanic 4 (Body-parser):
The body-parser converts raw price data into the somatic markers that populate state.body_signal. This is the tree's sensory organ:

body_parse(candle_sequence) -> f32:
    gut = weighted_sum(
        spread_speed * 0.3,      // how fast the spread is moving
        tick_asymmetry * 0.25,   // buy vs sell tick imbalance
        candle_body_ratio * 0.2, // body vs wick ratio
        volume_surge * 0.15,     // volume relative to recent
        acceleration * 0.1       // second derivative of price
    )
    return tanh(gut)  // squashed to [-1, +1]

This is mechanic 24's sensory input.

Mechanic 24 (Iowa Gambling Task -- body before mind):
The body_signal updates BEFORE the tree traversal completes. The somatic marker arrives at the leaf before the "rational" prediction:

Timeline for each tick:
    t+0ms:  new candle data arrives
    t+1ms:  body_parse() updates body_signal         <- BODY FIRST
    t+2ms:  tree traversal begins using new state
    t+5ms:  tree traversal completes, prediction made <- MIND SECOND

If body_signal and tree_prediction AGREE:    high confidence
If body_signal and tree_prediction DISAGREE: flag for review
    -> this is the "bad feeling" the Iowa Gambling Task measures
    -> the body knows before the tree does

Mechanic 16 (OPTIC equivalences):
Certain state configurations are equivalent under transformation. The tree recognizes these and maps them to the same node:

OPTIC transforms:
    O (octave):    price level is irrelevant; only ratios matter
    P (permutation): certain feature orderings are interchangeable
    T (transposition): shift in time does not change structure
    I (inversion):  mirror patterns map to same node with sign flip
    C (complement): complementary patterns share statistics

Implementation:
    Before traversal, normalize state by OPTIC equivalences.
    A rising wedge at 1.0850 and a rising wedge at 1.1200
    map to the SAME node. The tree does not waste branches
    on price level.

Mechanic 19 (Xenakis game theory):
At each branch point, the tree plays a game against the market. The market "chooses" which child to visit. The tree "chooses" how to weight its predictions:

Branch node with children A, B, C:

Market's "strategy": visit A 60%, B 30%, C 10% of the time
Tree's "strategy":   weight A's prediction by p_A, B by p_B, C by p_C

Minimax solution:
    Tree should weight predictions to minimize maximum regret
    This is NOT the same as weighting by visit frequency
    Rare-but-extreme child C may deserve disproportionate weight

    p_optimal = argmin_p max_market_strategy regret(p, market)

This prevents the tree from ignoring tail events just because they are rare.

Mechanic 28 (Rough volatility):
Volatility in the state vector is not smooth -- it has Hurst exponent H ~ 0.1, meaning it is much rougher than Brownian motion. The tree accounts for this:

state.volatility is computed using:
    realized_vol = sqrt(sum(log_returns^2) / N)

    BUT: weighted by fractional kernel with H = 0.1

    rough_vol(t) = integral_0^t  K(t-s, H) * dW(s)
    where K(t,H) = t^(H-0.5) / Gamma(H+0.5)

    This means: recent ticks matter MUCH more than old ticks
    Volatility can spike and revert faster than sqrt(t)
    The tree branches on rough vol, not smooth vol

This matters for branching: a node that splits on smooth vol would miss the fast-revert structure. Splitting on rough vol captures the actual dynamics.

G. Evolution and Lifecycle (The Seasons)

Mechanic 31 (Behavior trees):
The trading decision is a behavior tree that runs ON TOP of the data tree:

Behavior Tree (runs every tick):

    [SELECTOR]
    |
    +--[SEQUENCE: Check for signal]
    |   +-- Is confidence > threshold?
    |   +-- Does body_signal agree?
    |   +-- Is polyrhythm aligned? (M1+M5+M15 agree)
    |   +-- [ACTION: Generate trade signal]
    |
    +--[SEQUENCE: Manage existing position]
    |   +-- Is boundary detected?
    |   +-- Has tree re-traversed to different branch?
    |   +-- [ACTION: Adjust or close position]
    |
    +--[FALLBACK: Do nothing]
        +-- [ACTION: Continue observing, feed tree]

The behavior tree is the interface between the growing data tree and the trading decisions. The data tree grows continuously. The behavior tree queries it on demand.

VI. LIFECYCLE OPERATIONS

Pruning: When Leaves Die

PRUNE when:
    node.last_visited < (now - STALE_THRESHOLD)   // 90 days no visits
    AND node.committed == false                     // never committed
    AND node.visit_count < MIN_VISITS_TO_KEEP       // not enough history

Pruning is seasonal:
    Run full prune sweep once per month
    Dead leaves are removed
    Orphaned branches (all children pruned) collapse to parent
    This is autumn -- the tree sheds what it does not need

Mechanic 9 (Whale song revolution), revisited: Major pruning events -- where entire branches die -- correspond to market regime shifts. The tree records WHEN branches died. A dead branch from 2020 tells you something: "this regime existed once, it may return." The branch stays as deadwood (metadata preserved, statistics frozen) even after active pruning.

Committing: When Branches Become Trunk

COMMIT when:
    node.visit_count > COMMIT_THRESHOLD (e.g., 5000)
    AND node.outcomes.variance has been stable for > 60 days
    AND node's Kuramoto phase with neighbors > 0.9 (synchronized)

Once committed:
    node.committed = true
    node.state boundaries are FROZEN
    node can still update outcome statistics
    but it will NEVER split again
    it is now structural -- trunk or major branch

Mechanic 30 (Critical periods), revisited: Commitment IS a critical period closing. The node was plastic, now it is fixed. This is irreversible. The tree's committed structure is its "personality" -- the deep knowledge that does not change.

Why compounding:

  1. Committed nodes are free infrastructure. Once the trunk is committed, every new observation can skip directly to the relevant branch. Navigation cost decreases as the tree matures.

  2. Cross-branch pattern transfer. When branch A discovers a micro-pattern, the tree checks if branch B has the same pattern. If yes, both branches benefit from each other's data. The more branches, the more cross-pollination.

  3. Dead branches are knowledge too. A dead branch from 2020 tells the tree: "if volatility looks like THIS, the regime that USED to work here has ended." Dead branches prevent the tree from repeating mistakes.

  4. Rings enable temporal prediction. By year 10, the tree can identify "this state resembles ring 3 in that branch" -- meaning "this looks like what happened 7 years ago." The deeper the rings, the more historical analogies available.

Mechanic 28 (Rough volatility) + Mechanic 18 (Persistent homology) compound: Rough vol means the tree's branching on volatility captures fast dynamics. Persistent homology identifies which branches are structurally stable across scales. Together: the tree develops a volatility model that is both fast (rough) and robust (persistent). This takes 3-5 years to fully emerge but once present, it is the deepest part of the tree's knowledge.

XI. WHY THIS IS THE MOAT

The tree after 20 years of continuous growth is unreproducible. It cannot be copied, because:

  1. Path dependence means starting today produces a different tree than one started in 2006. The rings cannot be faked.

  2. Critical period closure means committed nodes were shaped by data that may never recur. A 2008 branch committed during the financial crisis contains knowledge that cannot be recreated from post-2008 data.

  3. The canopy shape -- the overall topology of the tree -- is an emergent property of 20 years of EUR/USD dynamics filtered through your specific edge. No one else has your sight-reading ability shaping which branches get sunlight.

  4. Dead branches are memory. The tree remembers regimes that ended. A competitor starting fresh has no dead branches, no memory of past market personalities.

  5. The tree IS the moat. Copying the code gives you a seed. The value is in the growth.
